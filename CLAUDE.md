# CLAUDE.md

本文件为 Claude Code (claude.ai/code) 提供在此仓库中工作的指导。

## 项目概述

这是一个使用 **webnovel-writer** 技能系统的中文网络小说创作项目。小说名为**《装乖千金：病娇大佬他自我攻略了》**，是一部现代言情/豪门恩怨小说，讲述了"装乖"女主和"自我攻略"男主的故事。

**当前进度**：第1卷（共3卷），已写25章（约58,100字）
**目标**：240章，约80万字

## 可用的斜杠命令

本项目使用专门的 webnovel-writer 技能：

| 命令 | 用途 |
|---------|---------|
| `/webnovel-write [章节]` | 撰写下一章或指定章节 |
| `/webnovel-review [范围]` | 审查章节质量（例如：`/webnovel-review 1-5`） |
| `/webnovel-plan [卷数]` | 生成某卷的详细大纲 |
| `/webnovel-query [主题]` | 查询项目状态、角色、剧情线索 |
| `/webnovel-resume` | 恢复中断的创作工作流 |
| `/webnovel-dashboard` | 启动只读网页仪表板 |
| `/webnovel-learn` | 将成功模式提取到项目记忆 |
| `/webnovel-analysis` | 分析小说或基于样本生成大纲 |

## 核心故事元素

**主角**：沈知意（24岁，流放6年后归来的真千金）
- **核心特质**："装乖"——表面柔弱温顺，内心锋利狠厉
- **金手指**：微表情洞察——识别谎言与情绪
- **人物弧线**：从防御性伪装者到自信的掌权人

**男主**：陆砚辞（28岁，陆氏集团总裁）
- **核心特质**："自我攻略"——看穿她的伪装却以为她暗恋自己
- **人物弧线**：从戏谑调侃到深情守护

**反派**：沈清瑶（占据女主位置13年的假千金）

## 硬性创作约束

来自 `idea_bank.json` 和 `大纲/总纲.md`：

1. **每章必须有钩子或爽点**（爽点/钩子）
2. **爽点密度**：至少每2章1个
3. **女主绝不圣母**——对敌人必须睚眦必报
4. **男主一眼看穿伪装**——没有长期的误会
5. **金手指限制**（微表情洞察）：
   - 每日限制：3次（午夜重置，不可累积）
   - 第1卷对男主无效（后续卷逐渐有效）
   - 过度使用触发反噬（头痛→短暂失明→暂时丧失能力）

## 关键状态文件

**`.webnovel/state.json`** - 恢复时务必先读取：
- `progress.current_chapter` - 最后撰写的章节
- `progress.total_written_words` - 当前字数
- `protagonist_state` - 角色物品栏、今日已使用能力、已收集情报
- `chapter_meta` - 每章的钩子、模式、结尾状态
- `review_checkpoints` - 已完成的审查及评分

**`.webnovel/idea_bank.json`** - 创意约束与提取的最佳实践

## 完整写作流程（必循）

### 阶段一：前置准备（Pre-writing）

撰写每一章前，必须完成以下检查：

1. **读取项目状态**
   ```bash
   # 读取 state.json 确认
   - progress.current_chapter - 当前章节号
   - protagonist_state.abilities.微表情洞察.used_today - 今日已用次数
   - protagonist_state.intel - 已收集的情报清单
   - chapter_meta[上一章] - 上一章的结尾状态（时间、地点、情绪）
   ```

2. **读取章节大纲**
   ```bash
   # 查看对应卷数的详细大纲，确认：
   - 本章目标、阻力、代价（目标-阻力-代价结构）
   - 时间锚点、倒计时状态
   - 金手指使用计划（每日最多3次）
   - 爽点类型（打脸/甜蜜/认知）
   - Strand类型（Fire/Constellation/Quest）
   - 章末钩子设计
   ```

3. **读取前情摘要**
   ```bash
   # 阅读最近3章的摘要 .webnovel/summaries/ch00XX.md
   - 关键情节进展
   - 人物关系变化
   - 未闭合的问题/伏笔
   ```

4. **验证硬性约束**
   - [ ] 爽点密度达标（本章或相邻章有爽点）
   - [ ] 金手指使用未超限（检查当日已用次数）
   - [ ] 符合主角"睚眦必报"人设
   - [ ] 男主对女主伪装保持"看穿+自我攻略"态度

### 阶段二：撰写执行（Writing）

使用技能命令启动完整写作流程：

```bash
/webnovel-write [章节号]
```

该命令自动执行以下子流程：

| 步骤 | 子Agent | 输出 |
|------|---------|------|
| Step 1 | context-agent | 情境包（前情+大纲+约束） |
| Step 2A | reviewer-agent | 一致性检查报告 |
| Step 2B | continuity-checker | 连贯性检查报告 |
| Step 3 | writer-agent | 初稿（2000-2500字） |
| Step 4 | ooc-checker + high-point-checker + pacing-checker | 质量检查报告 |
| Step 5 | polisher-agent | 润色稿 |
| Step 6 | data-agent | 实体提取、场景切片、摘要生成 |

### 阶段三：后置处理（Post-writing）

写作完成后，系统会自动：

1. **更新项目状态** - state.json 更新进度、角色状态
2. **生成章节摘要** - 保存至 .webnovel/summaries/ch00XX.md
3. **提取实体数据** - 记录新出场人物、地点、事件
4. **Git提交** - 自动提交全部文件

### 阶段四：批量审查（Review）

每完成5-10章，必须运行审查：

```bash
/webnovel-review [起始章]-[结束章]
```

审查流程包括：
- 设定一致性检查
- 人物OOC检查
- 爽点密度统计
- 节奏分析（Strand Weave）
- 追读力评估
- 生成审查报告于 `审查报告/第X-Y章审查报告.md`

### 紧急情况处理

**若写作流程中断**：
```bash
/webnovel-resume
```
该命令会读取 workflow_state.json 定位中断点并提供恢复选项。

**若需查看写作中的上下文**：
```bash
# 查看最新情境快照
ls .webnovel/context_snapshots/
```

```

```


## 重要提示

- 章节编号使用前导零（第0001章、第0017章）
- 每章目标字数2000-2500字
- 时间跟踪很关键——检查 `state.json` 中的当前故事日期
- 主角的"微表情洞察"使用记录在 `state.json` 的 `protagonist_state.abilities` 下
- `.webnovel/summaries/` 中的章节摘要用于情境连续性
